GitHub Workflow

GitHub is a platform that helps people solve problems by building software together. It's an online service that uses Git to provide code repositories.
Repositories are like a project's folder, containing all of the project files and revision history. They can have multiple collaborators and can either be public or private.
Here is a link to a page describing the GitHub flow

What is a commit?
A commit is an individual change to a file, for example when you add code to a branch.

What is a branch?
A branch is an alternate timeline where you can safely make changes to the software. It's a parallel version of a repository.
This allows you to work freely and make commits without affecting the "live" or master branch.

What is a pull request?
Pull requests let you show others the changes you are proposing so they can review and discuss them, before merging them with the master branch.
